## Use case 1
The first load of the widget shows an input to enter a free text intent 
The length of the text is controlled (using config setting) to prevent non meaningful AI calls

## Use case 2
persist filters intent activated before reaching the active categories for the widget
--> if the widget is active for a category, the active options can be enabled in the widget if the option exists in the 
products assigned to the category
--> if some filters are activated outside the category context, a button "View Filters" appears to apply the filters to the widget in bulk
--> if no filters are activate outside the category context, the button "View Filters" should not appear

## Use case 3
When the user starts to type some text in the intent input, a readiness bar show the number of characters to type for the widget to be
ready to query AI to provide recommendations

## Use case 4 
When the widget loads, a readiness bar shows the number of product matches available in the category with the current activated filters

### Use case 5
Each time the user selects an option for a filter, the readiness bar is updated showing the number of prooduct matches

### Use case 6
Each time the user selects an attribute filter, the options for the attribute are displayed. The options listed are cached.
If a new option is added or its label is updated, the cache needs to be cleared to take effect
